# i18n Gap Audit — pt-BR test findings (2026-07-06)

Triggered by testing the app in Portuguese and seeing mixed pt/en on every screen.
Root-caused to **three distinct bug classes**. Bug 1 accounts for ~90% of what's visible.

## Bug 1 — Untranslated placeholders (DOMINANT) 🔴

**~2,300 of 2,546 keys are English placeholders** in the non-English locale files —
wired correctly through `t()`, but the value is identical to English because the
actual translation step (Tolgee AI-translate) was never run.

| Locale | Translated (differs from en) | Placeholder (== en) | Missing |
|---|---|---|---|
| pt-BR | 243 | 2,263 | 40 |
| es | 241 | ~2,305 | (similar) |

The ~243 already-translated keys are older, pre-effort hand-translated ones. Every
key added during the B-wave extraction (Kids 585, Pregnancy 318, Garage 106, Cycle 80,
etc.) is an English placeholder.

**Why:** the extraction workflow inserts the English string as a placeholder in all
non-en files (so TypeScript compiles); Tolgee was meant to replace them but was never
connected.

**Fix:** generate real pt-BR + es translations for the ~2,300 placeholder keys and
write them into `lib/i18n/pt-BR.ts` + `es.ts`. (Chosen: translate via Claude directly,
skipping the Tolgee signup.)

## Bug 2 — Hardcoded strings the lint guard missed 🟠

The `no-literal-string` guard runs in `jsx-text-only` mode and its `files` glob had
gaps. Two sub-causes:

1. **`_layout.tsx` files excluded.** The radial "where to" menu in
   `app/(tabs)/_layout.tsx` has real copy — `"where to, Igor?"`, `"PICK A CORNER"`,
   `"little gifts"`, `"pass it on"`, corner labels — none extracted. (B5 excluded
   `_layout` files as "layout-only," which was wrong for this one.)
2. **Runtime-transformed / non-JSX-child literals.** e.g.
   `components/home/cycle/CycleJourneyRingFull.tsx` returns bare strings like
   `'rising day'` / `'follicular'` from a `switch`, later rendered and uppercased
   ("DRAG TO SPIN", "a rising day"). `jsx-text-only` can't see strings that aren't
   direct JSX children.

**Fix:** add `_layout` files with copy to the guard glob; extract those strings;
extract the phase-label/hint returns in the cycle-ring components.

## Bug 3 — Data-array / object-literal values 🟠

User-facing strings that live in data structures, not JSX, so `jsx-text-only` never
flagged them:
- Badge names: `"Social Butterfly"`, `"Mood Watcher"`, `"Emotion Tracker"`, etc.
- Growth-leap `signs` / labels in `lib/growthLeaps.ts` + `components/home/KidsHome.tsx`
- Grandma quick-chip labels in `components/chat/GrandmaTalk.tsx` (`"Sleep routine"`, …)
- Sleep-quality `tag`/`blurb` maps in KidsHome

**Fix:** extract the display labels from these arrays to keys (values referenced by key),
same pattern used for the onboarding option arrays in earlier waves.

## Priority

1. **Bug 1 first** — it's 90% of the visible gap and unblocks a genuinely trilingual
   test. One batch translation pass.
2. **Bug 2** — small, high-visibility (home + radial menu are the first screens seen).
3. **Bug 3** — lowest volume, mostly Kids-mode badge/leap detail.

## Guard hardening (prevent recurrence)

- Add `_layout.tsx` files (that carry copy) to the lint glob.
- Consider `mode: 'all'` on the guard for `lib/` data modules that hold display labels,
  or a targeted check for `label:/title:/name:` string literals in known data files.
- Note: the guard can never catch a *placeholder* (a key whose value happens to equal
  English) — that's what `i18n:check` + a new "untranslated placeholder" report should
  cover. Recommend adding a `npm run i18n:untranslated` script that lists keys where a
  locale value === en value.
